Project Overview
Project Name

Medicine Landing Experience

Vision

Create a premium storytelling website that showcases a multi-symptom medicine through an immersive scroll-based experience.

Unlike traditional pharmaceutical websites that present information statically, this experience should feel like an interactive product journey where the medicine bottle itself becomes the narrator.

Every animation must support the story rather than exist for visual decoration.

The final experience should feel comparable to Apple, Nothing, Linear, Stripe, and modern editorial storytelling websites while maintaining exceptional performance, accessibility, and SEO.

Continuous Motion Rules

Bottle should NEVER teleport.

Never disappear.

Never suddenly resize.

Everything should feel like one uninterrupted movement.

Motion Language

Allowed:

Translate
Rotate
Scale
Opacity

Mask reveals
Clip-path reveals (light usage)

Avoid:

Bounce
Elastic eases
Flashing
Shaking
Random particle explosions
Large parallax layers
Scroll hijacking
Long pin durations that trap the user
Section Transition Rules

Every section should overlap by roughly 15–20% of the viewport. The next chapter should begin to appear before the previous one fully exits. This creates a continuous narrative instead of abrupt cuts.

Performance Rules for Scroll

Every scroll interaction must follow these constraints:

Animate only transform and opacity.
Keep pinned sections short (around one viewport) to avoid scroll fatigue.
Reuse the same bottle asset throughout the experience.
Use CSS gradients and SVG masks instead of expensive particle systems where possible.
Destroy and recreate ScrollTrigger instances cleanly on resize.
Ensure smooth performance on mid-range mobile devices.
